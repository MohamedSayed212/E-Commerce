3. Branch workflow

Create a branch:

git checkout -b feature/navbar

Switch branches:

git checkout main
git checkout feature/navbar

Modern alternative:

git switch main
git switch -c feature/navbar

Update your branch from main:

git checkout main
git pull
git checkout feature/navbar
git merge main
4. Typical React task workflow
git checkout main
git pull
git checkout -b feature/add-user-card
npm install
npm run dev

After coding:

git status
git add .
git commit -m "Add user card component"
git push -u origin feature/add-user-card

Then open a Pull Request on GitHub.

5. Good commit messages

Use clear, simple messages:

git commit -m "Fix navbar mobile layout"
git commit -m "Add product card component"
git commit -m "Refactor auth form validation"

Avoid:

git commit -m "changes"
git commit -m "fix"
git commit -m "stuff"
6. Pull requests

A good PR should include:

What changed
Why it changed
Screenshots for UI changes
Testing notes

Example:

## What changed
Added responsive navbar for mobile and desktop.

## Why
Needed for the new landing page layout.

## Testing
Tested on Chrome at desktop and mobile widths.
7. Handling conflicts

When Git shows conflict markers:

<<<<<<< HEAD
old code
=======
new code
>>>>>>> feature-branch

Edit the file manually, keep the correct code, then:

git add .
git commit
8. Undo commands you should know

Undo unstaged file changes:

git checkout -- src/App.jsx

Modern:

git restore src/App.jsx

Unstage a file:

git restore --staged src/App.jsx

Change last commit message:

git commit --amend -m "Better commit message"

Delete local branch:

git branch -d feature/navbar

12. Junior developer habits

Before starting work:

git pull

Before pushing:

npm run lint
npm test
npm run build

Before opening PR:

Check your changed files.
Remove console logs.
Make sure the app runs.
Write a clear PR description.
Add screenshots for UI work.

Learn these first and you’ll be ready for most junior React team workflows.
